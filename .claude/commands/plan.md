$ARGUMENTS

Create an implementation plan for the described work:

1. **Understand**: Read relevant code and docs. Map the current architecture.
2. **Design**: Propose the approach. Identify affected files and interfaces.
3. **Break down**: Split into ordered tasks with clear checkpoints.
4. **Risk**: Note assumptions, unknowns, and potential blockers.
5. **Present**: Show the plan with estimated effort per task.

Do not start implementing. This is a planning step only. Wait for approval before proceeding.
