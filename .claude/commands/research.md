$ARGUMENTS

Research the codebase before making changes:

1. **Map**: Identify all files and modules related to the topic.
2. **Trace**: Follow the execution path. Understand how data flows.
3. **Conventions**: Note patterns, naming, error handling style.
4. **Dependencies**: List what depends on the code you'll change.
5. **Summarize**: Report your findings. Include file paths, key functions, and risks.

Use subagents (Task tool with Explore type) for parallel research across different areas.
Do not make any changes. This is research only.
